1. Discuss the scope of var, let, and const

Var:

Basically variable declared with var is both function scope & and global scope. That means we can access them before they are declared in our code. If a var variable is declared outside of any function, it becomes a global variable and accessible throughout our entire code. Possible to reassign and update.

Let:

Mainly let variables have block scope. They are only accessible within the block. If we declare let within a function, let variables are limited to that function's scope. Variables declared with let can be reassigned new values within their scope. This means we can change the value of a let variable as many times as we want within it's block.

Const:

const variables are also block-scoped, this means they are only accessible within the block of code where they are defined. It cannot be reassigned to a different value within the same scope.

2. Tell us the use cases of null and undefined

Null:

null represents an empty or missing value in many programming languages, including JavaScript. It's used to signify the absence of an object or a variable that should have a value but does not have value.

Undefined:

undefined is a special value in JavaScript that represents the absence of a defined value. When trying to access an object property that does not exist, then undefined occurs. When a function does not return a value explicitly, it returns undefined by default.

3. What do you mean by REST API?

API:

Basically API is the bridge of communication between different software components, enabling them to interact and exchange data with each other. For example if we order something in a restaurant where a waiter take orders from us and go to the kitchen and bring our ordered items for us. Here the waiter is like an API. Here the client and server are separate entities that communicate over a network.
